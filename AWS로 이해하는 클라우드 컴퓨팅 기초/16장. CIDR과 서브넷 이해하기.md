# 16장. CIDR과 서브넷 이해하기

## 이 장에서 말하고자 하는 것

앞 장에서 우리는 **VPC**를 배웠다.

VPC는 클라우드 안에서 사용하는  
하나의 **네트워크 공간**이다.

이 네트워크 안에는 여러 서버가 존재한다.

예를 들어 다음과 같은 서버가 있을 수 있다.

```
웹 서버
API 서버
DB 서버
캐시 서버
```

이 서버들이 서로 통신하려면  
각각을 구분할 수 있는 주소가 필요하다.

이때 사용하는 것이 **IP 주소**다.

---

# 1. 서버는 IP 주소로 서로를 구분한다

네트워크에서 장치들은  
IP 주소를 사용해 서로를 구분한다.

예를 들어 다음과 같은 구조를 생각해보자.

```
웹 서버  → 10.0.0.10
API 서버 → 10.0.0.20
DB 서버  → 10.0.0.30
```

웹 서버가 DB 서버와 통신하려면  
DB 서버의 IP 주소를 알아야 한다.

즉, IP 주소는  
**네트워크에서 장치를 식별하는 주소**다.

---

# 2. 그런데 IP 주소에는 충돌 문제가 있다

여기서 한 가지 문제가 있다.

만약 두 개의 서로 다른 네트워크가  
같은 IP 주소를 사용한다면 어떻게 될까?

예를 들어 두 회사가 모두  
다음과 같은 IP를 사용한다고 가정해보자.

```
10.0.0.10
```

두 네트워크가 연결되면  
어느 서버로 연결해야 하는지 구분할 수 없게 된다.

이런 문제를 방지하기 위해  
네트워크에서는 **사용할 IP 주소 범위를 미리 정한다.**

---

# 3. IP 범위를 표현하는 방법: CIDR

네트워크에서 사용할 IP 범위를  
표현하는 방식이 **CIDR**이다.

표기 방식은 다음과 같다.

```
IP주소 / 숫자
```

예를 들어 다음과 같은 표현이 있다.

```
10.0.0.0/16
```

이 의미는 다음과 같다.

* 네트워크 시작 주소: `10.0.0.0`
* `/16` → 네트워크 범위 크기

이 네트워크에서는 약 **65,536개의 IP 주소**를 사용할 수 있다.

즉,

```
10.0.0.0 ~ 10.0.255.255
```

범위의 주소를 사용할 수 있다.

---

# 4. VPC도 CIDR 범위를 가진다

AWS에서 VPC를 생성할 때  
가장 먼저 해야 하는 것이 있다.

바로 **VPC의 CIDR 범위를 정하는 것**이다.

예를 들어 다음과 같이 설정할 수 있다.

```
10.0.0.0/16
```

이 의미는

> 이 VPC에서는
> 10.0.0.0 ~ 10.0.255.255 범위의 IP를 사용한다

는 뜻이다.

VPC 안에 생성되는 서버들은  
이 범위 안에서 IP 주소를 할당받는다.

---

# 5. 하지만 모든 서버를 같은 네트워크에 두지는 않는다

VPC 안에 서버가 많아지면  
모든 서버를 하나의 네트워크에 두는 것은 관리가 어려워진다.

예를 들어 다음과 같은 서버들이 있다고 하자.

```
웹 서버
API 서버
DB 서버
캐시 서버
로그 서버
```

이 서버들이 모두 같은 네트워크에 있다면  
다음과 같은 문제가 발생할 수 있다.

예를 들어 웹 서버가 외부 공격을 받아  
네트워크 접근 권한이 노출되었다고 가정해보자.

이 경우 공격자는  
같은 네트워크에 있는 다른 서버에도 접근을 시도할 수 있다.

```
웹 서버 → DB 서버 접근 시도
웹 서버 → 캐시 서버 접근 시도
웹 서버 → 내부 서비스 접근 시도
```

모든 서버가 하나의 네트워크에 있다면  
이런 접근을 제한하기 어렵다.

---

# 6. 그래서 네트워크를 나누어 사용한다

이 문제를 해결하기 위해  
네트워크를 더 작은 단위로 나누어 사용한다.

예를 들어 다음과 같은 구조다.

```
네트워크 A
웹 서버

네트워크 B
API 서버
DB 서버
```

이렇게 나누면
다음과 같은 정책을 적용할 수 있다.

```
인터넷 → 웹 서버 허용
웹 서버 → API 서버 허용
API 서버 → DB 서버 허용
인터넷 → DB 서버 차단
```

즉,

* 외부 사용자는 웹 서버만 접근 가능
* 내부 서버는 직접 접근 불가
* 필요한 통신만 허용

이렇게 네트워크를 나누면  
보안과 관리가 훨씬 쉬워진다.

---

# 7. 서브넷이란 무엇인가

이처럼 하나의 네트워크를  
더 작은 네트워크로 나눈 것을 **서브넷(Subnet)** 이라고 한다.

예를 들어 다음과 같은 VPC가 있다고 하자.

```
10.0.0.0/16
```

이 네트워크를 다음과 같이 나눌 수 있다.

```
10.0.1.0/24
10.0.2.0/24
10.0.3.0/24
```

각각이 하나의 **서브넷**이다.

---

# 8. VPC와 서브넷 구조

개념적으로 보면 다음과 같은 구조다.

```
VPC 10.0.0.0/16
 ├ Subnet 10.0.1.0/24
 ├ Subnet 10.0.2.0/24
 └ Subnet 10.0.3.0/24
```

각 서브넷에는
서버들을 따로 배치할 수 있다.

---

# 9. 이 장의 핵심 정리

1. 서버는 IP 주소로 서로를 구분한다.
2. 네트워크에서는 사용할 IP 범위를 미리 정한다.
3. 이 IP 범위를 표현하는 방식이 CIDR이다.
4. VPC도 CIDR 범위를 가진다.
5. 서버가 많아지면 네트워크를 나누어 관리한다.
6. 하나의 네트워크를 나눈 것이 서브넷이다.
